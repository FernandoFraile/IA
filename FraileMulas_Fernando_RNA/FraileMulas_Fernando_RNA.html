Función MLPRegressor¶

Lembremos que unha rede neuronal est formada de tres compoentes principais: as neuronas, as funcins de activacin de cada unha delas e os pesos de cada unha das conexins entre neuronas.

Imos construir a nosa primera rede neuronal empregando a funcin MLPRegressor que proporciona o paquete de scikit-learn. Imos empregar a funcin de activacin loxstica (ou sigmoidal) activation='logistic'. Lembremos que o problema do prezo das vivendas un problema de regresin, no que tomamos como entrada un nico valor, a superficie do inmoble, e queremos obter como sada outro valor, o prezo estimado do mesmo. Polo tanto, o tamao das capas de entrada e sada da rede neuronal que modele o problema ser igual a 1, dicir, bstanos cunha neurona na entrada e outra na sada. Imos empregar tamn unha terceira capa entre a capa de entrada e a capa de sada, o que se coece como capa oculta, que polo de agora vai ter tres neuronas. Esta rede ten o seguinte aspecto:

Xa podemos empregar a clase MLPRegressor para crear a nosa primeira rede. A esta funcin pasmoslle o nmero de neuronas na capa oculta con hidden_layer_sizes. ### Erro cadrtico medio ###
def ecm(reais, predicion):
    m = len(reais)  # tamao do conxunto de datos
    erro = sum((predicion - reais)**2) / (2*m)
    return erro

Algo non funcionou? O que sucede que nosa rede neuronal non capaz de xeneralizar o coecemento. Este fenmeno cocese como underfitting, ou axuste insuficiente. O underfitting pode darse por mltiples causas, como o baixo nmero de mostras no noso conxunto de adestramento ou a mala eleccin das caractersticas que eliximos para representar o problema. Sen embargo, neste caso o motivo outro. O rendemento dunha rede neuronal pode verse moi deteriorado se empregamos os datos en bruto sen normalizar. Moitas veces o algoritmo non converxer antes do nmero mximo de iteracins permitidas. Normalizar non mis que comprimir ou extender os valores dunha variable para que estn acoutados nun rango desexado. Existen diferentes mtodos para normalizar os datos. Neste caso imos empregar o mtodo min-max para escalar os datos das variables superficie e prezo ao rango [0,1]. Simplemente, para cada valor concreto $a_i$ dunha variable $a$, o seu correspondente valor normalizado $z_i$ vir dado por: $$ z_i = \frac{a_i - \min(a)}{\max(a) - \min(a)} $$

En Python podemos implementar unha funcin que tome como entradas as mostras (de adestramento e test) dunha variable e nolas devolva escaladas:

In [10]:
### Normalizacin min-max ###
def norm_min_max(train, test):
    maximo = train.max()
    minimo = train.min()
    z_train = (train - minimo) / (maximo - minimo)
    z_test = (test - minimo) / (maximo - minimo)
    
    return z_train, z_test

O learning rate (coeficiente de velocidade de aprendizaxe) determina canto afecta a retropropagacin aos pesos en cada iteracin. Este coeficiente equivalente ao $\alpha$ que vimos nas prctica 3.1 de regresin. Nas clases tericas empregabamos a letra grega $\eta$ para referirnos a el.

Un learning rate elevado implica que durante o adestramento cada iteracin pode modificar fortemente os valores dos pesos ou, dito doutro xeito, o algoritmo "aprende" moito de cada un dos exemplos do dataset. Isto tamn significa que un s outlier pode afectar seriamente aos valores dos pesos e facer que o algoritmo diverxa e non chegue a un axuste de pesos ptimo. Pola contra, un coeficiente de aprendizaxe baixo significa que o algoritmo "aprende" menos de cada exemplo do dataset, as que a rede neuronal verase menos afectada por outliers, anda que se necesitarn mis exemplos e iteracins para que o algoritmo converxa e se obtea un bo modelo.

En MLPRegressor podemos axustar este coeficiente co parmetro learning_rate_init, que ata agora deixamos fixo a 0.0025. esta funcin contina o adestramento ata acadar max_iter ou converxer, polo que se empregamos un learning_rate_init inapropiado (moi elevado ou moi baixo), MLPRegressor non converxer e amosar un WARNING (nos exemplos anteriores sempre se amosaba o WARNING porque desactivbamos o early stopping mediante n_iter_no_change=float('inf')). ## Funcin de preprocesado do dataset
def preprocesar_dixitos(df):
    # Tomamos as 784 variables de entrada
    df_X = df.iloc[:, 1:]
    # Normalizamos os seus valores
    df_X = norm_num(df_X)
    
    # Tomamos a variable de saida
    df_Y = df.iloc[:, 0:1]
    # Convertmola ao tipo de dato category (pois son etiquetas que poden tomar un nmero delimitado de valores)
    df_Y = df_Y.astype("category")
    # Renomeamos a columna
    df_Y.columns = ['label']
    
    # Xuntamos as novas entradas normalizadas coas novas sadas codificadas
    novo_df = pd.concat([df_X, df_Y], axis=1)
    
    return novo_df

# Funcin para aplicar de a normalizacin
def norm_num(df):
    columnas_numericas = list(df.select_dtypes(include=[np.number]).columns.values)
    if len(columnas_numericas) > 0:
        # Aplicamos unha normalizacion min-max, coa diferencia de que xa sabemos que os valores van de [0, 255]
        df.loc[:, columnas_numericas] = df.loc[:, columnas_numericas] / 255.0

    return df

Pregunta 1

Tenemos 3 pesos para cada cada conexin que llega a las 3 nueronas de la capa oculta, y 3 pesos para cada conexin de estas neuronas con la neurona de la capa de salida, teniendo as 6. Pregunta 2

Si se tratasen los datos de test y train de igual forma se produciran cambios en el entrenamiento de la funcin, ya que se estaran utilizando los datos de test para normalizar. Por otro lado, no se puede normalizar todo y luego separar, ya que podramos estar selecionando datos aleatoriamente para train, pero a partir de un cierto valor (por ejemplo de las colas, en el primer y ltimo cuantil). En este caso la construccin del modelo sera muy poco real ya que la mayora de valores esperados no estaran dentro de los datos de train. Se normalizan de forma independiente ya que los conjuntos train y test pueden tener valores mjy distintos, por lo que la normalizacin sera distinta, ya que tendran un mximo y mnimo distinto. Recordemos que el parmetro mu de la normal es el "centro" de la campana, y se si tienen datos de mximo y mnimos distintos, ese mu va a ser diferente, por lo que se normaliza en funcin de lo mximos y mnimos de la variable train, para que los datos de test tenga el mismo parmetro mu (estn en la misma distribucin normal).
